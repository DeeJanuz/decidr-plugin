# Unreleased

- **feat**: Default-organization picker. Star button per row in `UI.orgPicker` persists a user-level default via the new `/api/me/preferences` endpoint, independent of the active-org switch. Star state lives alongside the active-row highlight so users can set a default on an org whose plugin token isn't connected yet.
- **feat**: Picker UI rebuilt to match the Companion app-launcher glassmorphism — pill trigger with building icon, backdrop-blur dropdown, uppercase "Organizations" caption, accent-ghost active row with checkmark slot, "Default" pill badge, "Connect" badge for no-token orgs, hover-reveal stars. Dropdown is now right-anchored to match the header's `justify-content: space-between` layout.
- **feat**: Single-org users now see the picker (the `orgs.length <= 1` guard was removed) so they can still set a persistent default.
- **feat**: New API client methods `API.getUserPreferences()`, `API.setDefaultOrg(orgId)`, `API.clearDefaultOrg()` bound to `/api/me/preferences`.
- **fix**: Two-phase dashboard/graph initial fetch. Phase 1 pulls organizations, plugin-org tokens, and user preferences; the renderer resolves the target org (`pushed > default-with-token`) and calls `switchOrg` if it differs from the currently-bound token, then Phase 2 runs the data `Promise.all` against the correct token. Without this, default-org preference was silently ignored on fresh mount and users saw an empty dashboard until manually clicking a row. Graph preflight lives outside `_fetchGraphData` so explicit user switches don't loop back to the default.
- **fix**: `API.autoInit` now rejects when Tauri returns "No stored token" for an explicitly targeted org instead of silently resolving with an empty token and triggering a 401 cascade. The `window.__decidrToken` fallback is preserved for non-targeted boots.
- **fix**: Picker row markup restructured into sibling buttons (option + star) to eliminate invalid nested-button markup and event bubbling.
- **fix**: Stray `;` in `theme.js` that terminated the picker CSS string mid-assignment has been removed — prior builds were shipping the picker with no styles applied to the lower half of the rules.
